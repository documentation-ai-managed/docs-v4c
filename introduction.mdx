## Quick Start

Get started in minutes. Follow these steps to set up your first space.

<Steps>

<Step title="Create Account" icon="user-plus">

Sign up at `https://v4c.example.com/register`. Verify your email.

</Step>

<Step title="New Space" icon="plus">

From dashboard, click "New Space". Name it "Project Docs".

```bash
# Example CLI setup (optional)
npx v4c-cli init "Project Docs"
```

</Step>

<Step title="Add Document" icon="file-plus">

Navigate to your space. Create "introduction.mdx" with frontmatter.

<CodeGroup tabs="MDX,Markdown">

````mdx
---
title: My Doc
description: First document.
---
## Hello World
Content here.
````

````markdown
# My Doc (Markdown)
First document content.
````

</CodeGroup>

</Step>

<Step title="Publish" icon="globe">

Save and publish. Share link: `https://docs.v4c.example.com/project/introduction`.

</Step>

</Steps>

## Brand Guidelines

Follow V4c branding for consistency. Primary color is `#3B82F6`.

<Callout kind="info">

Use `#3B82F6` for headers and accents. Pair with neutrals for readability.

Example CSS:

```css
:root {
  --brand-primary: #3B82F6;
  --brand-text: #1F2937;
}
```

</Callout>

<Expandable title="Full Palette" default-open="false">

| Color Name | Hex Code     | Usage                  |
|------------|--------------|------------------------|
| Primary    | `#3B82F6`   | Buttons, links         |
| Secondary  | `#1D4ED8`   | Hover states           |
| Neutral    | `#6B7280`   | Body text              |
| Background | `#FFFFFF`   | Main areas             |

Apply in themes via `/themes`.

</Expandable>
